Core Features

Everything you need to
automate email at scale

Built on Django with a powerful email_automation engine — designed for teams and enterprises.

Inbox Intelligence
Automatically reads Gmail and IMAP inboxes, filters noise, and identifies emails that need a response — powered by LLM relevance scoring.
RAG-Powered Replies
Enriches every AI reply with your own knowledge base via Postgres + pgvector. Answers are grounded in your actual documentation and FAQs.
Secure Multi-Tenancy
Full user isolation — per-user encrypted credentials (Fernet), scoped state store, tenant-prefixed vector DB, and audit logging built in.
Flexible Scheduling
Runs via APScheduler in-process or scales out with Celery + Redis. Periodic polling, manual triggers, and priority queues all supported.
Gmail OAuth + SMTP/IMAP
Connect via Google OAuth 2.0 or any SMTP/IMAP provider. Per-user token files, automatic refresh, and OAuth callback handling included.
Dashboard & Audit Trail
Real-time dashboard showing pending queue, sent replies, skipped emails, and full audit logs — all scoped per user account.
Workflow

How MailPilot works

Six steps matching the live automation pipeline—from inbox poll to reply and per-user state.

1
Poll inbox
Gmail API (recent threads) or IMAP INBOX on a schedule, manual “Run poll”, or IMAP IDLE for faster SMTP inboxes.
2
Keyword prefilter
Optional SERVICE_KEYWORDS filter runs first—non-matching mail is skipped before any LLM call.
3
RAG lookup
If your KB is configured: embed subject+body and pull top chunks from Postgres pgvector (JSON upload).
4
AI relevance & reply
One LLM call returns relevance, confidence, and reply text; must pass your RELEVANCE_THRESHOLD.
5
Draft or auto-send
REPLY_MODE=draft saves the reply for dashboard review; send delivers immediately via Gmail API or SMTP.
6
State & queue
Per-user processed state avoids duplicates; the dashboard queue shows sent, draft, and ignored activity (account audit in admin).

FAQ

Common questions

Quick answers about setup, routing, knowledge base, and billing. Still stuck? Use the contact form on this page.

What does MailPilot do?

MailPilot connects to your Gmail or IMAP inbox, filters incoming mail with keywords and AI relevance, grounds replies in your knowledge base, and can send or draft responses automatically.

Does it reply to every email?

No. You set keywords and a relevance threshold so only service-related messages are handled. Unrelated mail is ignored. Start with draft mode to review before enabling auto-send.

Gmail or SMTP/IMAP—which should I use?

Gmail OAuth is the fastest setup for Google Workspace or personal Gmail. Use SMTP + IMAP for other providers. You can test the connection in Setup before going live.

How does the knowledge base work?

Upload JSON or text, or crawl your website. MailPilot chunks content, stores embeddings in PostgreSQL with pgvector, and pulls the best matches into each AI reply.

Is there a free trial?

Yes. The Starter plan includes a one-month trial with a monthly send limit so you can connect an inbox and test relevance filtering before paid plans launch.

How is my data protected?

Credentials and API keys are stored encrypted per account. See our Privacy Policy for more detail.

When will paid plans be available?

Pro and Custom plans are marked Coming soon on pricing. Sign up for the free trial now; contact us for agency or high-volume pricing.
